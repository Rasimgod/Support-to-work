git init: Эта команда создает новый репозиторий Git в текущем каталоге. После выполнения этой команды Git начинает отслеживать изменения файлов в этом каталоге.

git clone: Клонирует существующий репозиторий Git из удаленного источника на ваш компьютер. Вы указываете URL репозитория, и Git создает копию репозитория на вашем компьютере.

git add: Добавляет изменения в индекс для последующего коммита. Вы указываете файлы или каталоги, которые хотите добавить, и Git начинает отслеживать эти изменения.

git commit: Фиксирует изменения, добавленные в индекс, и создает новый коммит. Вы указываете сообщение коммита, которое описывает ваши изменения.

git status: Отображает состояние рабочего каталога и индекса. Вы узнаете, какие файлы были изменены, добавлены в индекс или исключены из него.

git diff: Показывает различия между рабочим каталогом и индексом или между коммитами. Вы можете видеть, какие строки были добавлены, изменены или удалены.

git log: Отображает историю коммитов. Вы видите список всех коммитов в репозитории, включая информацию о каждом коммите, такую как автор, дата и сообщение.

git branch: Показывает список веток, а также создает, удаляет и переключается между ними. Вы можете создать новую ветку для разработки новой функции или исправления ошибок.

git checkout: Переключается между ветками, восстанавливает файлы из коммитов или веток. Вы также можете создать новую ветку или переместиться на определенный коммит.

git merge: Объединяет изменения из одной ветки в другую. Вы указываете ветку, из которой хотите объединить изменения, и Git автоматически объединяет их с текущей веткой.

git remote: Управляет удаленными репозиториями. Вы можете добавить новый удаленный репозиторий, удалить существующий или просмотреть список удаленных репозиториев.

git fetch: Получает изменения из удаленного репозитория, но не применяет их к текущей ветке. Вы можете получить последние изменения без автоматического объединения.

git pull: Получает изменения из удаленного репозитория и автоматически объединяет их с текущей веткой. Это сочетание команд git fetch и git merge.

git push: Отправляет изменения из локального репозитория в удаленный. Вы указываете ветку, которую хотите отправить, и Git загружает ваши изменения на сервер.

git stash: Сохраняет временные изменения, чтобы переключиться на другую ветку или решить конфликты. Вы можете временно спрятать незакоммиченные изменения и затем вернуться к ним позже.
